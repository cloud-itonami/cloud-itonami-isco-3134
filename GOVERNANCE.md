# Governance

## Decision Making

Decisions about the direction, scope, and features of this project are made by the maintainers of the cloud-itonami organization.

## Issue Triage

Issues are reviewed and triaged by maintainers. Community members are welcome to comment and provide additional context.

## Pull Requests

All pull requests must:
- Pass all tests
- Include tests for new functionality
- Have a clear description of the changes
- Be reviewed and approved by at least one maintainer before merging

## Code Review Standards

Code reviews focus on:
- **Correctness** — does the code do what it's supposed to do?
- **Safety** — do hard invariants hold? Are process-control operations correctly blocked?
- **Testability** — are tests clear and comprehensive?
- **Maintainability** — is the code clear and idiomatic Clojure?

## Release Process

Releases are tagged with semantic versioning (vMAJOR.MINOR.PATCH).
